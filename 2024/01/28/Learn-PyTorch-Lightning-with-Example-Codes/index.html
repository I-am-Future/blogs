Learn PyTorch-Lightning with Example Codes

Posted on 2024-01-28 Edited on 2024-02-08

In this blog, we’ll learn pytorch-lightning framework with some simple example codes. I believe the code will significantly help you understand it.

As for a deep learning training & testing process, we need to define the following components:

  • Data (by pl.LightningDataModule)
  • Model (by pl.LightningModule)
  • Configs (pytorch-lightning have utilities for CLI. But here we use another package called configargparse, which support both config file and CLI configs)

1. Data

We can define a pl.LightningDataModule which implements procedures to initialize the Dataset and DataLoader.

Main components:

  • __init__(): The constructor. You can save the configurations (hyper-parameters) to the class here.
  • setup(): Will be called before fit, validate, test, and predict. You can initialize the datasets as class members here. Note that it has an argument stage.
  • prepare_data(): Will be called only on Master process. In distribute training, setup() would be called on all processes. If you only want to only do one thing (e.g., downloading data), please implement it in the prepare_data().
  • train_dataloader(), val_dataloader(), test_dataloader(), predict_dataloader(). Will be called during the corresponding phase for DataLoader. It should return a DataLoader for that stage.
